# telega_stalker

Каркас Telegram-игры в стиле S.T.A.L.K.E.R. на `python3 + aiogram`.

## Что уже реализовано

1. Приветствие и регистрация:
   - `Привет, сталкер! Какое у тебя прозвище?`
   - создание персонажа: прозвище + пол
2. Выбор группировки:
   - `Долг`
   - `Свобода`
3. Меню персонажа:
   - **Инвентарь**: статы, снаряга, деньги, энергия, вещи, топливо
  - **Профиль**: карточка персонажа с визуализацией сталкера, локации и текущей снаряги
  - используется классический стиль аватара (без переключения)
  - PNG-аватары подхватываются автоматически:
    - `assets/avatars/stalker_t1..t4.png`
    - fallback: `assets/avatars/stalker_default.png`
   - визуальная "ава по снаряге": образ персонажа меняется вместе с улучшением брони/шлема
   - система скинов по прогрессу снаряги:
     - Новичок (0+)
     - Ветеран (4+)
     - Тяжелый штурмовик (8+)
     - Легенда Зоны (13+)
  - **Торговец**: покупка/продажа, ручная экипировка оружия/брони, ремонт, грузовик, топливо
  - **Задания**: 4 типа сложности с максимумом шанса
    - шанс учитывает не только снарягу, но и амуницию (патроны/аптечки)
    - у каждого задания есть обязательный расход амуниции
     - Легко — до 90%
     - Сложно — до 80%
     - Тяжело — до 70%
     - Невозможно — до 60%
4. Война за точки:
   - точки ресурсов (деньги в казну группировки)
   - базы
   - точки интереса (ускоряют прибытие, если под контролем вашей стороны)
   - бой по формуле шанса: `сила_отряда / (сила_отряда + сила_NPC)`
5. Энергия и переходы:
   - действия тратят энергию
   - энергия постепенно восстанавливается
   - можно пить энергетики
   - грузовик ускоряет перемещения, но требует топливо
6. Задания и амуниция:
   - шанс теперь считается от базы сложности + силы снаряги + бонусов амуниции
   - учитываются патроны и аптечки в инвентаре
   - для задания нужно минимальное количество патронов (в зависимости от сложности)
   - при запуске задания патроны и аптечки расходуются
7. Отрядные рейды:
   - общий рейд нескольких игроков одной группировки
   - создание рейда по локации, присоединение и запуск лидером
   - расчет боя по участникам: оружие, броня, криты, урон, ранения
8. Динамические события Зоны:
   - генерация событий на карте
   - модификаторы силы NPC по локациям
   - динамический спавн/усиление NPC через события
9. Расширенная экономика:
   - склад группировки (сдача/выдача ресурсов)
   - внутренний аукцион группировки (лоты, покупка, отмена)
   - контрабандные рейсы с риском и наградой

## Быстрый старт

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

В `.env` задай:

```env
BOT_TOKEN=твой_токен_бота
DB_PATH=/data/stalker_game.db
```

Используй только одну базу:
- `DB_PATH=/data/stalker_game.db`
- это единый файл, в котором хранится весь прогресс игроков по `telegram_id`.

Также можно явно задать backup-файл:
- `SNAPSHOT_PATH=/data/stalker_game.backup.json`

Запуск:

```bash
python3 run.py
```

## Деплой на Railway

Важно: в `main` у тебя сейчас только `README.md`.  
Если Railway подключен к `main`, он не увидит Python-файлы и покажет ошибку вида:
`Railpack could not determine how to build the app`.

Что сделать:

1. В Railway выбери ветку с кодом бота:  
   `cursor/aiogram-stalker-game-d161`
2. Добавь переменную окружения:
   - `BOT_TOKEN` — токен Telegram-бота
3. Укажи `DB_PATH=/data/stalker_game.db`
4. Укажи `SNAPSHOT_PATH=/data/stalker_game.backup.json`
5. Команда запуска уже поддержана через `start.sh` и `railway.json`.

Локальная проверка start-скрипта:

```bash
chmod +x start.sh
./start.sh
```

## Структура

- `app/bot.py` — хендлеры aiogram и FSM регистрации
- `app/storage.py` — SQLite-хранилище (персонажи, фракции, локации)
- `app/game_logic.py` — квесты, торговля, энергия, война, переходы
- `app/keyboards.py` — reply/inline-клавиатуры
- `app/profile_card.py` — генерация PNG-карточки профиля игрока
- `app/avatar_render.py` — генерация визуальной авы сталкера по уровню снаряги
- `assets/avatars/stalker_t1..t4.png` — готовые шаблоны аватаров под 4 тира экипировки
- `run.py` — точка входа

## Важно для Railway (сохранение персонажа)

- Обязательно укажи `DB_PATH=/data/stalker_game.db`, чтобы база лежала в persistent volume.
- Не используй несколько разных `.db` файлов одновременно: бот должен работать с одним `DB_PATH`.
- У каждого игрока есть отдельный ID-адрес вида `STK-<telegram_id>`, он показывается в инвентаре.
- Бот автоматически делает резервную копию прогресса в JSON рядом с БД (например, `/data/stalker_game.backup.json`).
- При запуске, если база пустая, бот пытается восстановить персонажей/фракции/локации из этого backup-файла.
- Фоновая синхронизация backup выполняется каждые 5 минут.
- Доступно пополнение баланса за Telegram Stars:
  - кнопка `⭐ Пополнить` в основном меню
  - курс: `1 звезда = 10 RU`
  - поддерживаемые пакеты: 1 / 5 / 10 / 25 звезд
  - есть кнопка `⭐ Другое количество` для ввода своего числа звезд вручную

## Новые механики (v2)

- **Отрядные рейды**:
  - раздел `🪖 Рейды` в меню
  - лидер создает рейд на логово, бойцы той же группировки присоединяются
  - запуск рейда только лидером, минимум 2 участника
  - успешный рейд на логово с силой NPC от 25 дает награду артефактами (1-2 за запуск)
- **Границы для войны**:
  - штурм в разделе `⚔ Война` доступен только при наличии минимум 5 живых бойцов в группировке
- **Углубленный бой**:
  - учитываются оружие/броня каждого бойца
  - есть критические попадания
  - есть ранения (снижение HP) по итогам рейда и контрабанды
- **События Зоны и динамический NPC-спавн**:
  - раздел `🛰 События`
  - при генерации события меняется модификатор локации и сила NPC
  - события живут ограниченное время и влияют на войну/рейды/контрабанду
- **Расширенная экономика**:
  - раздел `🏦 Экономика`
  - склад группировки (сдача/выдача ресурсов)
  - аукцион внутри группировки (создание лотов/покупка/отмена)
  - контрабандные рейсы с риском и наградой

## Обновления Торговли (новые предметы и экипировка)

- Оружие и броня, купленные у торговца, теперь **попадают в инвентарь**.
- Экипировка делается вручную:
  - `🛒 Торговец` → `🟢 Покупка` → `🛡 Снаряжение` → `Экипировать оружие/броню`.
- При экипировке:
  - выбранный предмет снимается из инвентаря;
  - ранее экипированный предмет возвращается в инвентарь.
- Новые предметы брони доступны в разделе `Покупка брони` и продаются в `Продажа брони`.
- Продажа экипированного оружия/брони учитывает **износ**: чем ниже прочность, тем ниже итоговая цена выкупа.
